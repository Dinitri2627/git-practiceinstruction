## Глава 1: Введение и Начальная Настройка

Создание аккаунта на GitHub и установка Git на ваш компьютер.

### 1.1. Регистрация на GitHub

GitHub — это веб-сервис для хостинга IT-проектов и их совместной разработки. Основан на системе контроля версий Git.

1.  Перейдите на сайт [github.com](https://github.com/).
2.  Нажмите кнопку **"Sign up"**.
3.  Следуйте инструкциям на экране: введите вашу почту, придумайте пароль и имя пользователя.
4.  Подтвердите свой аккаунт через письмо, которое придет на указанную почту.

### 1.2. Установка Git

Git — это программа, которая устанавливается на ваш компьютер и выполняет контроль версий.

1.  Перейдите на официальный сайт [git-scm.com](https://git-scm.com/).
2.  Скачайте установочный файл для вашей операционной системы (Windows, macOS, Linux).
3.  Запустите установщик. Для большинства пользователей подойдут настройки по умолчанию.
4.  После установки откройте терминал и проверьте, что Git установлен корректно.

**Пример проверки версии:**
\`\`\`bash
$ git version
git version 2.51.0.windows.1
\`\`\`
**Необходимо:** вам нужно ввести свои данные в Git, чтобы коммиты подписывались вашим именем.
\`\`\`bash
$ git config --global user.name "Имя"
$ git config --global user.email "Почта@example.com"
\`\`\`
## Глава 3: Работа с Удаленным Репозиторием 

Эта глава объясняет, как взаимодействовать с удаленными серверами (GitHub). 

### 3.1. Базовое взаимодействие с удаленным репозиторием

Для начала необходимо создать пустой репозиторий в GitHub

**`git remote add <имя> <URL>`**: Связывает ваш локальный репозиторий с удаленным.
- `<имя>` — это короткое имя-псевдоним для URL, по стандарту используется `origin`.
- `<URL>` — адрес вашего репозитория на GitHub.


*Пример:*
\`\`\`bash
$git remote add origin https://github.com/YourUsername/git-guide-project.git
\`\`\`

**`git push -u <имя_сервера> <имя_ветки>`**: Отправляет  ваши коммиты из локальной ветки на удаленный сервер.
- Флаг -u (или --set-upstream) устанавливает связь между вашей локальной и удаленной веткой. В следующий раз достаточно будет написать просто git push.

\`\`\`bash
$git push -u origin master
\`\`\`
**`git push --all origin`**: Отправляет все коммиты и ветки на удаленный сервер.

\`\`\`bash
git push --all origin
\`\`\`
**`git pull`**: Загружает изменения с удаленного сервера и автоматически пытается слить их с вашей текущей локальной веткой. 

\`\`\`bash
git pull origin master
\`\`\`

**`git clone <URL>`**: Создает полную локальную копию уже существующего удаленного репозитория, включая всю его историю.

### 3.2. Работа с форками

Fork — это ваша личная, полная копия чужого репозитория на вашем аккаунте GitHub. Вы являетесь владельцем этой копии и можете делать с ней все что угодно: создавать ветки, делать коммиты. Эти изменения не затрагивают оригинальный проект.


Pull Request — это ваше официальное предложение авторам оригинального проекта принять изменения, которые вы сделали в своем форке.

\`\`\`bash
$ git clone https://github.com/Ban049/Russian_ban_words
Cloning into 'Russian_ban_words'...
remote: Enumerating objects: 19, done.
remote: Counting objects: 100% (3/3), done.
remote: Compressing objects: 100% (3/3), done.
remote: Total 19 (delta 0), reused 0 (delta 0), pack-reused 16 (from 1)
Receiving objects: 100% (19/19), 29.71 KiB | 239.00 KiB/s, done.
Resolving deltas: 100% (5/5), done.
\`\`\`


1. Откройте страницу вашего форка на GitHub.
2. Перейдите на вкладку "Pull requests" и нажмите "New pull request".
3. Убедитесь, что вы предлагаете слияние из вашей ветки в main ветку оригинального репозитория.
4. Напишите осмысленный заголовок и описание для вашего PR, объясняя, что и зачем вы сделали.
5. Нажмите "Create pull request".
